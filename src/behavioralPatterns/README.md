# 행위 패턴 (Behavioral Patterns)

> 📁 상위 문서: [프로젝트 루트 README](../../README.md)

행위 패턴은 **객체 간의 책임 분배와 상호작용(알고리즘, 흐름)** 을 다루는 디자인 패턴이다.
객체들이 어떻게 협력하고 통신하는지를 정의하여, 결합도를 낮추면서 유연한 동작을 구성한다.

## 진행 현황
> ⏳ 현재 폴더 구조만 준비되어 있고 **예제 소스는 아직 작성 전**이다.
> 각 패턴의 예제가 추가되면 구조 패턴과 동일하게 역할 주석과 개별 README를 정리할 예정이다.

## 패턴 목록 (작성 예정)

| 패턴 | 한 줄 요약 | 경로 | 상태 |
|------|-----------|------|------|
| **커맨드 (Command)** | 요청을 객체로 캡슐화해 실행·취소·큐잉을 유연하게 처리 | [`command`](./command) | 예정 |
| **인터프리터 (Interpreter)** | 언어의 문법을 클래스로 표현해 문장을 해석 | [`interpreter`](./interpreter) | 예정 |
| **이터레이터 (Iterator)** | 컬렉션의 내부 구조를 노출하지 않고 순차 접근 제공 | [`iterator`](./iterator) | 예정 |
| **중재자 (Mediator)** | 객체 간 직접 참조를 없애고 중재자를 통해 상호작용 | [`mediator`](./mediator) | 예정 |
| **메멘토 (Memento)** | 객체의 상태를 저장했다가 나중에 복원(undo) | [`memento`](./memento) | 예정 |
| **옵서버 (Observer)** | 상태 변화를 구독자들에게 자동으로 통지 | [`observer`](./observer) | 예정 |
| **상태 (State)** | 내부 상태에 따라 객체의 행동을 바꿈 | [`state`](./state) | 예정 |
| **전략 (Strategy)** | 알고리즘군을 캡슐화해 런타임에 교체 | [`strategy`](./strategy) | 예정 |
| **템플릿 메서드 (Template Method)** | 알고리즘의 뼈대를 정의하고 세부 단계를 서브클래스가 구현 | [`templateMethod`](./templateMethod) | 예정 |
| **방문자 (Visitor)** | 객체 구조를 바꾸지 않고 새로운 연산을 외부에서 추가 | [`visitor`](./visitor) | 예정 |
