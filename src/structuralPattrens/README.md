# 구조 패턴 (Structural Patterns)

> 📁 상위 문서: [프로젝트 루트 README](../../README.md)

구조 패턴은 **클래스와 객체를 조합해 더 큰 구조를 만드는** 방법을 다루는 디자인 패턴이다.
객체 간의 관계를 유연하고 효율적으로 구성하여, 기능 확장과 유지보수를 쉽게 만드는 데 목적이 있다.

## 패턴 목록

| 패턴 | 한 줄 요약 | 문서 |
|------|-----------|------|
| **어댑터 (Adapter)** | 서로 호환되지 않는 인터페이스를 변환기로 연결해 기존 코드 수정 없이 함께 사용 | [adaptor/README.md](./adaptor/README.md) |
| **브릿지 (Bridge)** | 추상화와 구현을 분리(합성으로 연결)해 두 계층을 독립적으로 확장 | [bridge/README.md](./bridge/README.md) |
| **컴포지트 (Composite)** | 개별 객체와 복합 객체를 동일한 인터페이스로 다뤄 트리 구조를 재귀 처리 | [composite/README.md](./composite/README.md) |
| **데코레이터 (Decorator)** | 원본을 장식자로 겹겹이 감싸 기존 코드 수정 없이 기능을 동적으로 추가 | [decorator/README.md](./decorator/README.md) |
| **파사드 (Facade)** | 복잡한 서브시스템을 하나의 통합 창구로 감싸 단순한 인터페이스 제공 | [facade/README.md](./facade/README.md) |
| **플라이웨이트 (Flyweight)** | 공통의 무거운 데이터를 공유(캐싱)해 대량 객체 생성 시 메모리 절약 | [flyweight/README.md](./flyweight/README.md) |
| **프록시 (Proxy)** | 실제 객체를 대신하는 대리자를 두어 접근을 제어(지연 생성 등) | [proxy/README.md](./proxy/README.md) |

## 패턴별 핵심 비교

| 패턴 | 해결하는 문제 | 핵심 기법 |
|------|--------------|-----------|
| 어댑터 | 인터페이스 불일치 | Target 구현 + Adaptee 합성으로 호출 변환 |
| 브릿지 | 기능/구현 조합 폭증 | 추상화가 구현을 합성으로 참조("다리") |
| 컴포지트 | 부분-전체 계층 표현 | Leaf/Composite가 같은 인터페이스 + 재귀 |
| 데코레이터 | 기능의 동적 확장 | 자기 타입을 감싸 체인 구성 후 위임 |
| 파사드 | 복잡한 사용법 단순화 | 서브시스템을 감춘 통합 인터페이스 |
| 플라이웨이트 | 대량 객체의 메모리 낭비 | 내재적 상태 공유 + 외재적 상태 분리 |
| 프록시 | 실제 객체 접근 제어 | 같은 인터페이스의 대리 객체가 위임 |

> 각 패턴의 구성 요소(역할), 구조 다이어그램, 실행 흐름 예시는 위 표의 개별 문서에서 확인할 수 있다.
